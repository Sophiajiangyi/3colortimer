# 时间记录 PWA — 需求规格文档

## 一、项目背景
个人自用的时间记录工具，用于统计每天在「学习(英语)」「工作」「摆烂(娱乐视频)」三类活动上花费的时间，帮助了解自己的时间分配情况。

## 二、技术方案
- **形态**：PWA（渐进式网页应用），非原生App，不需要上架应用商店
- **使用方式**：手机Chrome打开网址 → 点击"添加到主屏幕" → 桌面出现图标，全屏使用，体验接近原生App
- **平台**：安卓手机为主
- **部署**：GitHub 仓库 + GitHub Pages（免费静态托管，HTTPS，满足PWA安装条件）
- **数据存储**：IndexedDB（浏览器本地数据库），数据永久保存在用户手机本地，不联网、不需要账号、不做云同步
- **多任务注意事项**：计时开始时必须立刻把"开始时间戳"写入 IndexedDB（而非只存内存变量），确保用户切换到其他App（如英语学习App）期间，即使该网页标签页被系统回收，切回后仍能根据时间戳正确计算时长

## 三、数据结构

每条记录（存入 IndexedDB）：

```
{
  id: 唯一标识,
  category: "学习" | "工作" | "摆烂",
  startTime: 时间戳（毫秒）,
  endTime: 时间戳（毫秒）,
  durationMinutes: 自动计算 = (endTime - startTime) / 60000,
  source: "timer" | "manual"   // 区分：实时计时 还是 手动补录
}
```

## 四、核心功能（MVP）

### 1. 首页 — 三个分类计时按钮
- 「学习」「工作」「摆烂」三个独立的开始/结束按钮
- **同一时刻只能有一个分类在计时中**（避免逻辑混乱），正在计时的按钮需高亮显示 + 实时显示已进行时长
- 点击"开始"立即将 startTime 写入本地存储；点击"结束"计算 duration 并生成一条记录

### 2. 手动补录
- "+" 按钮打开表单：选择分类 + 填写开始时间和结束时间（或直接填时长）
- 用于事后补记（例如忘记点计时）
- 补录的记录 source 标记为 "manual"，与实时计时的记录做区分（列表里可以有小标记区分，但不影响统计口径）

### 3. 记录列表（今日）
- 按时间倒序展示今天的每条记录：分类、开始-结束时间、时长、记录来源
- 支持**编辑**（改分类/时间）和**删除**单条记录

### 4. 今日统计
- 三类时间的占比展示（饼图或分段进度条）
- 每类：总时长、次数、平均单次时长

### 5. 本周统计
- 近7天的柱状图，三个分类分组或堆叠显示
- 用于观察一周内的时间分配趋势变化

### 6. 数据导出
- "导出" 按钮，生成包含全部历史记录的 CSV 文件（字段：分类、开始时间、结束时间、时长、记录来源）
- 导出后可通过手机分享/保存，作为数据备份手段（提醒用户定期导出，因为数据仅存本地，清除浏览器数据/恢复出厂设置会导致丢失）

## 五、明确不做的功能（控制复杂度）
- 不设定"每日目标/达标提示"等带有评判色彩的设计
- 不加备注/标签字段（只做时长统计，不做内容细分）
- 不做多设备云同步（每台设备数据独立）
- 不做通知提醒/推送

## 六、设计原则
- 界面保持简洁，避免类似《指尖时光》那种功能繁杂的设计
- 数据呈现保持中性客观（如实展示时长和占比），不做"未达标""摆烂太多"等评判性提示语

## 七、后续可能的迭代方向（暂不实现）
- 每类时间在一天中的时段分布图
- 多周/多月的长期趋势对比
- 数据云同步（如果未来有跨设备需求）
